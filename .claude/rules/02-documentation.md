# Documentation Rules

Purpose: keep documentation concise, traceable, and non-duplicative.

Scope: Markdown, docs, and governance paths.

Rules:

- Distinguish canonical, derived, reference, proposed, superseded, and archived documents.
- Update `governance/DOCUMENT_REGISTRY.md` when creating or changing document authority.
- Do not present assumptions as facts.
- Avoid empty templates and speculative documents.
- Prefer concise boundary READMEs for reserved directories.
- Link to canonical governance instead of copying it.
